|------|------|
| 行业 | 企业培训领域 |

## 二、当前 AI 应用现状

### 2.1 已构建的应用类型

**创作型应用**
- 课件创作
- 访谈
- 知识问答
- 题本创作
- 培训项目规划

**演绎型应用**
- 老师授课
- 对练
- 评测
- 销售等

### 2.2 模型使用情况

| 项目 | 内容 |
|------|------|
| 使用的模型厂商 | OpenAI / Google / Claude / Qwen / Doubao / Kimi / Zhipu |
| 是否自建模型 | 否，但针对测评场景有过微调 |
| 部署方式 | 使用厂商云服务 API |
| 模型选择策略 | 功能和质量优先，成本其次 |
| RAG 使用 | 使用了 RAG，但不是向量方向 |

### 2.3 自建技术设施

- **模型网关**：自建，基于 OpenAI 协议
- **服务对象**：主要服务于外部客户
- **用户规模**：早期爬升阶段，目标是百万用户

---

## 三、核心痛点与挑战

| 痛点类别 | 具体问题 |
|----------|----------|
| **协议兼容性** | 自建模型网关在适配多家厂商时遇到协议兼容问题 |
| **成本压力** | 随着用户规模增长，多厂商 API 调用成本较大 |
| **延迟问题** | 实时对练、授课等场景对延迟要求极高，部分场景延迟难以控制在 5s 以内，影响用户体验 |
| **稳定性问题** | 经常遇到限流及安全限制问题 |
| **算力资源** | 企业没有 GPU 卡资源，不考虑模型的私有化部署 |
| **团队能力** | 50 人技术团队，重点不在模型训练、部署、优化方面 |

---

## 四、合作期望优先级

按重要性排序：

1. **A. 算力成本优化** - 更低成本的推理/训练算力
2. **C. 技术支持** - Agent 研发、RAG、Prompt 工程等最佳实践分享
3. **E. 联合创新** - 共同探索新的 AI 应用场景
4. **B. 模型能力** - 更强的企业培训领域专用模型
5. **D. 基础设施服务** - 统一模型网关、部署平台等
6. **F. 市场合作** - 共同拓展企业培训市场客户

---

## 五、合作形式期望

- ✅ **技术合作**：技术交流、最佳实践分享、联合技术攻关
- ✅ **商务合作**：优惠算力包、联合解决方案、客户推荐
- ✅ **战略伙伴**：深度绑定、联合产品开发、共同市场拓展

## 六、其他关键信息

### 6.1 服务私有化部署

- 会有服务的私有化部署需求（适配企业已有模型）
- 客户通常自己有模型，或可提供开源模型部署
- 这部分不是主要业务

### 6.2 对云厂商网关服务的态度

- 有一定兴趣，但需要明确切换的价值主张和成本分担方式

---

## 七、需要阿里云证明的能力

### 7.1 需要对方提供的数据

| 企业关注点 | 需要阿里云证明 |
|----------|----------------|
| 算力成本优化 | 具体的成本对比数据，至少 30% 以上优势 |
| Agent 研发支持 | 企业培训领域的成功案例和最佳实践 |
| 模型能力 | 通义千问在我们场景下的实测效果 |
| 基础设施 | 与我们现有架构的兼容性和迁移成本 |
| 延迟优化 | 实时对练场景下 5s 内延迟的实际保证 |

### 7.2 可能的合作切入点（待验证）

以下方向需要阿里云提供具体方案和 PoC 验证：

1. **算力成本**：需证明 PAI-EAS 相比当前 API 的成本优势
2. **Agent 能力**：需证明百炼平台能提升我们的开发效率
3. **模型微调**：需证明测评场景下模型效果的显著提升
4. **延迟优化**：需证明实时场景下的延迟保证
5. **联合创新**：需证明联合解决方案的市场价值
