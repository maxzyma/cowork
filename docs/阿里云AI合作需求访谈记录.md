# 阿里云 AI 合作需求访谈记录

**访谈日期**：2026-01-16
**访谈目的**：与企业共创 AI 应用需求清单，作为与阿里云当面沟通的提纲

---

## 一、企业基本情况

| 项目 | 内容 |
|------|------|
| 行业 | 企业培训领域 |
| 规模 | 约 500 人 |
| AI 技术团队 | 约 50 人 |
| 业务类型 | 面向 B 端为主，也有 C 端业务 |

---

## 二、当前 AI 应用现状

### 2.1 已构建的应用类型

**创作型应用**
- 课件创作
- 访谈
- 知识问答
- 题本创作
- 培训项目规划

**演绎型应用**
- 老师授课
- 对练
- 评测
- 销售等

### 2.2 模型使用情况

| 项目 | 内容 |
|------|------|
| 使用的模型厂商 | OpenAI / Google / Claude / Qwen / Doubao / Kimi / Zhipu |
| 是否自建模型 | 否，但针对测评场景有过微调 |
| 部署方式 | 使用厂商云服务 API |
| 模型选择策略 | 功能和质量优先，成本其次 |
| RAG 使用 | 使用了 RAG，但不是向量方向 |

### 2.3 自建技术设施

- **模型网关**：自建，基于 OpenAI 协议
- **服务对象**：主要服务于外部客户
- **用户规模**：早期爬升阶段，目标是百万用户

---

## 三、核心痛点与挑战

| 痛点类别 | 具体问题 |
|----------|----------|
| **协议兼容性** | 自建模型网关在适配多家厂商时遇到协议兼容问题 |
| **成本压力** | 随着用户规模增长，多厂商 API 调用成本较大 |
| **延迟问题** | 实时对练、授课等场景对延迟要求极高，部分场景延迟难以控制在 5s 以内，影响用户体验 |
| **稳定性问题** | 经常遇到限流及安全限制问题 |
| **算力资源** | 企业没有 GPU 卡资源，不考虑模型的私有化部署 |
| **团队能力** | 50 人技术团队，重点不在模型训练、部署、优化方面 |

---

## 四、合作期望优先级

按重要性排序：

1. **A. 算力成本优化** - 更低成本的推理/训练算力
2. **C. 技术支持** - Agent 研发、RAG、Prompt 工程等最佳实践分享
3. **E. 联合创新** - 共同探索新的 AI 应用场景
4. **B. 模型能力** - 更强的企业培训领域专用模型
5. **D. 基础设施服务** - 统一模型网关、部署平台等
6. **F. 市场合作** - 共同拓展企业培训市场客户

---

## 五、合作形式期望

- ✅ **技术合作**：技术交流、最佳实践分享、联合技术攻关
- ✅ **商务合作**：优惠算力包、联合解决方案、客户推荐
- ✅ **战略伙伴**：深度绑定、联合产品开发、共同市场拓展

## 六、其他关键信息

### 6.1 服务私有化部署

- 会有服务的私有化部署需求（适配企业已有模型）
- 客户通常自己有模型，或可提供开源模型部署
- 这部分不是主要业务

### 6.2 对云厂商网关服务的态度

- 有一定兴趣，但需要明确切换的价值主张和成本分担方式

---

## 七、阿里云 AI 能力匹配度分析

### 7.1 可匹配的能力

| 企业需求 | 阿里云对应能力 |
|----------|----------------|
| 算力成本优化 | GPU 实例、PAI-EAS 推理服务、弹性计费、抢占式实例 |
| Agent 研发支持 | 大模型服务平台百炼（可视化构建 Agent）、MCP 服务 |
| 模型能力 | 通义千问系列（Qwen3）、模型微调服务 |
| 基础设施 | PAI 平台、统一模型部署与管理 |
| 技术交流 | 阿里云技术团队最佳实践分享 |

### 7.2 潜在合作点

1. **算力成本**：PAI-EAS 模型推理服务、GPU 实例的弹性计费
2. **Agent 能力**：百炼平台的可视化 Agent 构建能力
3. **模型微调**：针对企业培训领域（测评场景）的模型定制
4. **延迟优化**：PAI-Prime 的训练/推理优化技术
5. **联合创新**：基于企业培训场景的联合解决方案
